# ona_security_policy (Resource)

Ona security policy for environment runtime controls. Attach the resulting policy through organization policy settings to make it the default for new environments.

For product context, see [Guardrails](https://ona.com/docs/ona/guardrails/overview), [Veto](https://ona.com/docs/ona/guardrails/veto), and [Organization policies](https://ona.com/docs/ona/organizations/policies/overview).

## Example Usage

```terraform
resource "ona_security_policy" "baseline" {
  organization_id = "00000000-0000-0000-0000-000000000000"
  name            = "baseline"

  spec {
    ports {
      default_effect = "allow"

      rule {
        range_from = 22
        range_to   = 22
        effect     = "block"
      }
    }

    executables {
      default_effect = "allow"

      rule {
        path   = "/usr/bin/nc"
        effect = "audit"
      }
    }

    files {
      default_effect  = "allow"
      default_actions = ["read", "write"]

      rule {
        path    = "/etc/shadow"
        actions = ["read"]
        effect  = "block"
      }
    }

    block_devices {
      default_effect = "block"
    }

    data {
      default_effect = "allow"

      rule {
        source {
          file = "/workspace/secrets.env"
        }
        destination {
          host = "example.com"
        }
        effect = "block"
      }

      rule {
        source {
          integration = "00000000-0000-0000-0000-000000000001"
          selector    = "example-organization/private-repository"
        }
        destination {
          host = "api.example.com"
        }
        effect = "audit"
      }
    }
  }
}
```

## Schema

### Required

- `name` (String) Security policy name shown in Ona. Must be between 1 and 80 characters.
- `organization_id` (String) Organization ID that owns the security policy. Changing this value replaces the policy.

### Optional

- `spec` (Block, Optional) Runtime security controls enforced for environments using this policy. (see [below for nested schema](#nestedblock--spec))

### Read-Only

- `created_at` (String) Time when the security policy was created.
- `id` (String) Security policy ID.
- `updated_at` (String) Time when the security policy was last updated.
