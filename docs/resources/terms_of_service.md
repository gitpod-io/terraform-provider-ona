# ona_terms_of_service (Resource)

Singleton Ona organization Terms of Service configuration and current Markdown version. Organization scope is resolved from the configured provider token. Destroying this resource disables the Terms of Service requirement but does not delete immutable version history because the Ona API has no delete operation. Publishing or changing Markdown currently requires a user personal access token because Ona records the publishing user on each immutable version.

For product context, see [Terms of Service](https://ona.com/docs/ona/organizations/terms-of-service).

## Example Usage

```terraform
resource "ona_terms_of_service" "example" {
  enabled  = true
  markdown = file("${path.module}/terms.md")
}
```

## Schema

### Required

- `enabled` (Boolean) Whether organization members must accept the current Terms of Service version.

### Optional

- `markdown` (String) Desired Markdown body for the current Terms of Service version. When configured and changed, Ona publishes a new immutable version. When omitted, Terraform manages only `enabled` and preserves the API's current Markdown in state after read or import. Must be at most 32000 characters.

### Read-Only

- `current_version` (Number) Current Terms of Service version number, if one exists.
- `current_version_created_at` (String) RFC3339 timestamp for the current Terms of Service version, if one exists.
- `current_version_created_by_user_id` (String) User ID recorded by Ona as publisher of the current Terms of Service version.
- `current_version_id` (String) ID of the current immutable Terms of Service version, if one exists.
- `id` (String) Authenticated organization ID for the managed Terms of Service singleton.
