# ona_users (Data Source)

Lists Ona users visible to the configured token in its organization. Suspended or departed users can require organization-admin access.

For product context, see [Organization roles](https://ona.com/docs/ona/organizations/organization-roles).

## Example Usage

```terraform
data "ona_users" "active_members" {
  search   = "example.com"
  statuses = ["active"]
  roles    = ["member"]
}

locals {
  active_members_by_id = {
    for user in data.ona_users.active_members.users : user.user_id => user
  }
}
```

## Schema

### Optional

- `roles` (Set of String) Organization roles to include. Supported values are `admin` and `member`.
- `search` (String) Case-insensitive substring search over user names and email addresses. A UUID search matches that user ID exactly. Blank input is ignored.
- `statuses` (Set of String) User statuses to include. Supported values are `active`, `suspended`, and `left`.
- `user_ids` (Set of String) UUIDs of users to include. The Ona API accepts at most 25 values.
